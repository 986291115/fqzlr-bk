## 二、实施方案

### 方案选择

**说说页面**：需要适配——让编辑器直接操作 `src/content/moments/*.md` 文件（类似 ChangelogEditor 的模式），而不是操作 TS 配置文件。这样与页面展示数据源一致。

**更新日志页面**：直接复制参考项目的 ChangelogEditor，稍作适配即可。

### 步骤 1：复制并适配 ChangelogEditor

**目标**：为更新日志页面添加完整的编辑功能

**操作**：
1. 从参考项目复制 `ChangelogEditor.svelte` 到当前项目
2. 修改：移除 `EditToolbar` 的 `mountTo=".page-header-toolbar-slot"` 属性
3. 修改：添加侧边栏事件监听（`edit:sidebarModeChange`、`edit:sidebarSaveDraft`、`edit:sidebarSubmit`、`edit:sidebarCancel`）
4. 修改：添加 `$effect` 发送 `edit:hasChanges` 事件
5. 在 `src/pages/changelog.astro` 中引入 `<ChangelogEditor client:only />`
6. 确保 DOM 结构中有 `id="cl-timeline"` 和 `id="cl-filter-bar"` 供编辑器收集数据

**数据格式验证**：
- 当前项目 changelog 的 schema：`version`, `date`, `time?`, `type`, `description`
- 参考项目 ChangelogEditor 使用的字段：`slug`, `version`, `date`, `type`, `description`, `body`
- **差异**：当前项目没有 `body` 字段（只有 description），参考项目有 `body`（详情内容）
- **处理**：使用 description 作为主要内容，body 可选

### 步骤 2：实现 MomentsEditor（基于 markdown 文件）

**目标**：为说说页面添加完整的编辑功能，直接操作 markdown 文件

**操作**：
1. 基于 ChangelogEditor 的多文件编辑模式，创建 MomentsEditor.svelte
2. 数据结构：
   - 字段：`author`, `avatar`, `pinned`, `published`, `images`, `tags`, `location`, `device`, `content`
3. 从 DOM 中收集现有说说数据（SSR 渲染的内容）
4. 支持增删改查
5. 编辑模式下显示内联编辑表单
6. 保存为草稿，提交到 GitHub
7. 每个说说一个 markdown 文件，路径：`src/content/moments/{date-slug}.md`
8. 在 `src/pages/moments.astro` 中引入 `<MomentsEditor client:only />`

**说说编辑器功能列表**：
- 添加新说说
- 编辑现有说说（内容、图片、标签、位置、置顶）
- 删除说说
- Markdown 内容编辑（简单文本编辑器即可）
- 图片 URL 列表（逗号或换行分隔，最多9张）
- 置顶排序
- 草稿保存
- 批量提交

### 步骤 3：更新 EditPostButton.svelte 的页面配置

## 五、风险与注意事项

1. **说说编辑器工作量较大**：需要从零实现基于 markdown 多文件的编辑器，需要仔细处理字段映射
2. **DOM 数据收集**：编辑器需要从 SSR 渲染的 HTML 中收集初始数据，需要确保 DOM 结构中的 data 属性正确
3. **图片字段复杂**：说说的 images 字段支持 ImageMetadata 和 string 混合类型，编辑时需要统一处理为字符串 URL
4. **提交后刷新**：提交成功后刷新页面以显示最新内容
