# 编辑按钮功能修复计划

## 一、问题分析

当前项目的编辑按钮存在以下问题：

1. **显示范围不对**：每个页面左侧都显示写文章按钮，实际上只有 `/posts/`、`/categories/`、`/archive/` 这三个页面需要显示"写新文章"按钮
2. **文章列表页不需要编辑按钮**：`/posts/` 页面不需要"编辑当前文章"按钮，只有点进具体文章页面才显示
3. **编辑功能不完整**：点击编辑按钮后，缺少密钥导入、提交、保存草稿、批量提交、清除全部等功能按钮
4. **页面顶部多余的编辑按钮**：部分页面右上角有重复的编辑按钮需要移除

## 二、参考项目架构

参考项目 `C:\Users\fqzlr\Documents\trea\zzwork\my-blog` 的实现方式：

### 2.1 两种编辑模式

| 模式 | 适用页面 | 实现方式 |
|------|---------|---------|
| **跳转编辑** | 文章写作/编辑 | 跳转到 `/write/` 页面，使用 `WriteEditor.svelte` |
| **内联编辑** | 关于页、友链页、导航页等 | 直接在当前页面编辑，使用 `EditToolbar` + 各页面编辑器 |

### 2.2 侧边栏按钮逻辑

- **写作区块**：包含"编辑当前X"按钮 + "写新文章"按钮
- **显示条件**：只有特定页面显示（文章相关页面）
- **文章详情页**：显示"编辑当前文章" + "写新文章"
- **文章列表/分类/归档页**：只显示"写新文章"，不显示编辑按钮

### 2.3 EditToolbar 完整功能

点击编辑按钮展开后显示的完整按钮列表：
1. 取消/重置
2. 保存草稿（显示当前页面草稿数徽章）
3. 导入密钥 / 已认证（导入 .pem 私钥文件）
4. 提交
5. 批量提交（显示总草稿数徽章）
6. 清除全部（显示清除数徽章）
7. 添加（可选）
8. 帮助（使用说明弹窗）

### 2.4 事件通信机制

- 侧边栏按钮通过 `CustomEvent` 与页面编辑器通信
- 使用 `pageKey` 区分不同页面类型
- 事件类型：`edit:sidebarModeChange`、`edit:sidebarCancel`、`edit:sidebarSaveDraft`、`edit:sidebarSubmit`

## 四、具体实现步骤

### 步骤 1：重写 EditPostButton.svelte

**目标**：侧边栏写作区块包含完整的编辑工具栏功能

**实现要点**：
- 集成 EditToolbar 的全部功能（密钥导入、草稿、提交、批量提交等）
- 根据页面 URL 自动判断显示内容：
  - 文章详情页 (`/posts/xxx`)：显示"编辑当前文章"按钮 + "写新文章"按钮
  - 文章列表/分类/归档页 (`/posts/`、`/categories/`、`/archive/`)：只显示"写新文章"按钮
  - 其他内联编辑页面（关于、友链、导航等）：显示对应页面的编辑按钮 + "写新文章"按钮
- 点击编辑按钮后，展开完整工具栏（取消、保存草稿、导入密钥、提交、批量提交、清除全部、帮助）
- 文章页面的编辑按钮点击后跳转到 `/write/?path=xxx`（加载当前文章进行编辑）

### 步骤 2：调整 sidebarConfig.ts

**目标**：控制 editPostButton 的显示范围

**实现要点**：
- 所有页面都显示编辑按钮（因为每个页面都有自己的编辑功能）
- "写新文章"按钮在所有页面都显示（方便快速写文章）
- "编辑当前X"按钮根据页面类型动态显示/隐藏

### 步骤 3：确保各编辑器响应侧边栏事件

**目标**：每个编辑器都能正确响应侧边栏的编辑事件

**需要验证的编辑器**：
- DaohangEditor (pageKey: "daohang") - 导航页
- FriendsEditor (pageKey: "friends") - 友链页
- MarkdownPageEditor (pageKey: 可配置) - 关于页、赞助页等
- WriteEditor (pageKey: "write") - 写文章页

### 步骤 4：文章编辑跳转逻辑

**目标**：点击文章详情页的"编辑当前文章"按钮，跳转到写文章页面并加载当前文章

**实现要点**：
- 在 EditPostButton 中检测当前是否为文章详情页
- 获取当前文章的 slug/path
- 点击编辑按钮后跳转到 `/write/?path={文章路径}`
- WriteEditor 已支持通过 URL 参数加载文章

4. **向后兼容**：不破坏现有已经工作的编辑器功能
