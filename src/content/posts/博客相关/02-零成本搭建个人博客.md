## 整体方案

这套方案的核心思路：用 **Firefly**（基于 Astro 框架的开源博客主题）+ **Cloudflare Workers** 免费托管，全程零成本。

你不需要买服务器，不需要备案，不需要写代码（会改配置文件就行）。最终得到一个速度不错的个人博客站点。

技术栈简单说一下：

- **Astro**：一个静态网站生成框架，生成的都是纯 HTML，加载很快
- **Firefly**：基于 Astro 的博客主题，开箱即用，界面好看
- **Cloudflare Workers**：Cloudflare 的边缘计算服务，免费额度够个人用
- **GitHub**：代码托管，配合 Cloudflare 实现自动部署

## 第一步：安装环境

需要装四个东西，都是免费的：

**Node.js（版本 >= 22）**

去 [nodejs.org](https://nodejs.org/) 下载 LTS 版本安装。装完打开终端验证：

```bash
node -v    # 应该显示 v22.x.x 或更高
npm -v     # 能看到版本号就行
```

如果命令无效，重启终端试试，还不行就重启电脑。

**pnpm**

```bash
npm install -g pnpm
pnpm -v    # 验证
```

网络超时的话切手机热点试试。

**Git**

去 [git-scm.com](https://git-scm.com/download/win) 下载安装，一路默认就行。

**VS Code**（可选但推荐）

去 [code.visualstudio.com](https://code.visualstudio.com/) 下载安装，后面改配置文件、写文章都用它。

## 第二步：Fork 源码仓库

1. 打开 Firefly 官方仓库：[github.com/CuteLeaf/Firefly](https://github.com/CuteLeaf/Firefly)
2. 点右上角 **Fork** 按钮
3. 等几秒，跳转后你就有了自己的仓库

Fork 的意思就是把别人的项目复制一份到你自己的 GitHub 账号下，你可以随意修改，不会影响原作者。

## 第三步：本地跑起来

打开终端，依次执行：

```bash
# 克隆你的仓库（把地址换成你自己的）
git clone https://github.com/你的用户名/Firefly.git
cd Firefly

# 安装依赖
pnpm install

# 启动本地预览
pnpm dev
```

等 10-30 秒后终端会显示 `http://localhost:4321`，浏览器打开就能看到博客了。

这时候可以改改配置文件，比如站点名称、描述之类的，保存后页面会自动刷新。

改完之后别忘了创建 `wrangler.toml` 文件（Cloudflare 部署需要），在项目根目录新建：

```toml
name = "firefly"
compatibility_date = "2026-05-02"

[assets]
directory = "./dist"

[vars]
NODE_VERSION = "22"
```

`compatibility_date` 改成当天日期。

## 第四步：推送到 GitHub

先配置 Git 身份（只需设置一次）：

```bash
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub邮箱"
```

然后提交代码：

```bash
git add .
git commit -m "初始化我的博客"
git push
```

## 第五步：Cloudflare 自动部署

这是最关键的一步。配好之后，以后每次 `git push`，Cloudflare 都会自动帮你构建和上线。

1. 登录 [Cloudflare 控制台](https://dash.cloudflare.com/)
2. 左侧菜单找到 **Workers 和 Pages**
3. 点 **创建应用程序** -> 选择 **连接到 Git**
4. 授权 GitHub，选中你 Fork 的 Firefly 仓库
5. 配置构建命令：
   - **Build command**：`pnpm build`
   - **Deploy command**：`npx wrangler deploy`
6. 点 **部署**，等 1-3 分钟

构建成功后会给你一个 `xxx.workers.dev` 的临时域名，打开就能看到你的博客了。

## 第六步：绑定自己的域名

如果你有注册域名（参考第1期教程），可以绑定到 Cloudflare：

1. 在 Cloudflare Worker 项目里，找到 **触发器** -> **自定义域**
2. 添加你的域名
3. Cloudflare 会生成一个 CNAME 记录值
4. 去你的域名管理平台（比如 Spaceship），添加一条 CNAME 解析记录，指向 Cloudflare 给的值

DNS 生效需要 10 分钟到 24 小时不等，之后访问你的域名就能看到博客了。

## 写在最后

整个过程不需要写代码，核心操作就是 Fork 仓库、改配置、Cloudflare 关联 GitHub。部署完成后，写文章就是在 `src/content/posts/` 目录下新建 Markdown 文件，push 到 GitHub 就自动上线。
